cbr::V4L2Driver::V4L2Driver ( std::string  device = "/dev/video0",
uint32_t  n_buf = 4 
)
explicit

v4l2 driver for capturing image streams

This driver is flexible in terms of management: a user can supply a callback that is called on frames containing new buffers, and also custom allocation and de-allocation functions for allocating said buffers.

If no allocation functions are provided malloc()/free() is used.

Parameters
devicethe v4l object (default /dev/video0)
n_bufthe number of buffers to use (default 4)
